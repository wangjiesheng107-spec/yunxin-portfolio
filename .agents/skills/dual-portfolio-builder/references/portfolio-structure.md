# Portfolio Structure

## Web Version

The web version should be persuasive and interactive.

Recommended sections:

1. Hero
   - name
   - positioning
   - short intro
   - metrics
   - CTA buttons

2. About
   - education
   - career direction
   - work style

3. Experience
   - company
   - role
   - time
   - responsibilities
   - achievements

4. Projects
   Each project should show:
   - problem
   - role
   - method
   - result
   - tools
   - screenshots if available

5. Skills
   Use grouped tags:
   - HR
   - Product
   - AI tools
   - Data / workflow
   - Communication

6. Contact
   - email
   - GitHub
   - portfolio PDF
   - optional social links

## PDF Version

The PDF version should be more structured and less interactive.

Recommended order:

1. Cover
2. Profile
3. Core strengths
4. Experience
5. Projects
6. Skills
7. Contact

Rules:

- Do not overcrowd pages.
- Use strong headings.
- Use short paragraphs.
- Use project cards.
- Avoid long animation-dependent content.
