# Style Analysis Rubric

Use this rubric when `references/` contains screenshots, style images, website references, PDF references, or layout samples.

## Extraction Rule

Extract design principles, not identity.

Do not copy logos, brand names, original text, copyrighted illustrations, exact visual identity, or pixel-by-pixel layouts.

## Analyze

## Visual Direction

Describe the mood in practical words: professional, minimal, editorial, tech-oriented, corporate, warm, energetic, calm, premium, experimental, or playful.

## Color Palette

Extract:

- Primary color
- Secondary color
- Background color
- Surface color
- Text color
- Muted text color
- Accent color
- Border color

Prefer approximate hex values, then adapt them so the portfolio has its own identity.

## Typography

Extract:

- Heading scale
- Body scale
- Label/caption style
- Font weight contrast
- Line height
- Letter spacing
- Whether the reference feels geometric, editorial, humanist, technical, or system-native

Use accessible web fonts or system fonts unless the project already includes fonts.

## Layout

Extract:

- Grid style
- Section spacing
- Maximum content width
- Hero composition
- Card density
- Whitespace rhythm
- Mobile stacking behavior

## Components

Extract:

- Button shape and hierarchy
- Tag style
- Metric card style
- Project card style
- Timeline style
- Navigation treatment
- Contact/footer treatment

## Visual Details

Extract:

- Border radius
- Border weight
- Shadow softness
- Gradient usage
- Decorative shapes
- Icon style
- Image treatment

Avoid dominant one-note palettes. Keep the final portfolio readable, ownable, and suitable for HR/Product/AI applications.

## Output

Create `design-system.md` with:

```text
# Design System

## Visual Direction
## Color Palette
## Typography
## Spacing
## Cards
## Buttons
## Tags
## Data Cards
## Web Layout Rules
## PDF Layout Rules
## Do Not Copy
```
